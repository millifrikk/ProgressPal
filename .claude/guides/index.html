When you end a session, changes are automatically added to the changelog!

📊Session Management

Core Commands

/session-start

Begin a new development session with tracking

/session-start "auth-feature"
/session-start "fix-bug" --type=bugfix --priority=high

/session-update

Add progress updates with metrics

/session-update "Added JWT tokens"
/session-update "Database issue" --blocker

/session-current

View current session status

/session-current
/session-current --verbose

/session-end

End session with comprehensive summary

/session-end
/session-end "Phase complete"

/session-list

View all sessions with analytics

/session-list
/session-list --type=feature

/session-analytics

View productivity analytics

/session-analytics
/session-analytics --last-30-days

Session Types

Feature Refactor Bugfix Research Deployment

Update Types

Progress Update (Default)

/session-update "Implemented user authentication"

Blocker Update

/session-update "Cannot connect to database" --blocker

Decision Update

/session-update "Chose PostgreSQL over MySQL" --decision

Milestone Update

/session-update "Authentication complete" --milestone

📝Changelog System

Changelog Commands

/changelog-add

Add entry to changelog

/changelog-add "Added PDF export"
/changelog-add "Fixed bug" --category=fixed

/changelog-view

View changelog with filters

/changelog-view
/changelog-view --unreleased

/changelog-release

Create version release

/changelog-release 0.4.0
/changelog-release --auto

Change Categories

Added Changed Fixed Security Deprecated Removed

Automatic Integration

When you complete objectives and end a session, the changelog is automatically updated with your achievements!

🔄Example Workflows

Feature Development

# Start feature session
/session-start "user-dashboard" --type=feature --priority=high

# Regular updates
/session-update "Created dashboard layout"
/session-update "Added data visualization"
/session-update "All tests passing" --milestone

# End session (auto-updates changelog)
/session-end

# View what was added
/changelog-view --unreleased

Bug Fixing

# Start bugfix session
/session-start "fix-memory-leak" --type=bugfix --priority=critical

# Track investigation
/session-update "Reproduced issue"
/session-update "Found leak in WebSocket" --decision
/session-update "Fix verified" --milestone

# End and document
/session-end

Creating a Release

# Review unreleased changes
/changelog-view --unreleased

# Check session analytics
/session-analytics --summary

# Create release
/changelog-release 0.4.0

# Git commands (suggested)
git tag -a v0.4.0 -m "Release 0.4.0"
git push origin v0.4.0

✨Best Practices

Session Management

  • ✅ Start sessions with clear objectives
  • ✅ Update every 30-60 minutes
  • ✅ Use specific update types (blocker, decision, milestone)
  • ✅ Always end sessions properly for analytics
  • ✅ Review session analytics weekly

Changelog Writing

  • ✅ Write from user perspective
  • ✅ Be specific but concise
  • ✅ Group related changes
  • ✅ Include issue/PR references
  • ✅ Review before releases

Good Changelog Examples

✅ Good

  • "Added PDF export with custom formatting options"
  • "Fixed dashboard crash when viewing special characters"
  • "Improved API response time by 50%"

❌ Bad

  • "Fixed bug"
  • "Updated code"
  • "New feature"

💡Pro Tips

Session Templates: Use templates for consistency - they're in .claude/templates/
Metrics Tracking: Check /session-metrics during long sessions to monitor progress
Auto-Detection: Use /changelog-add --auto-detect to analyze git changes
Session Linking: Continue work with --continues=previous-session-id
